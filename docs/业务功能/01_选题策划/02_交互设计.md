# AI 交互设计方案 - 基础加载状态 (Simplified Loading)

## 1. 背景
为了快速落地并保持界面简洁，暂时不采用复杂的流式思维链展示。

## 2. 交互流程设计

### 2.1 状态定义
1.  **请求发起 (Request)**
    - 用户点击“生成”按钮。
    - 前端发起常规 HTTP POST 请求。

2.  **等待中 (Loading)**
    - **界面表现**：
        - 按钮变为不可点击。
        - 展示一个标准的**旋转 Loading 图标** (Spinner)。
        - 提示文案：`正在思考中...` (或 `AI 正在为您生成选题...`)。
    - **超时处理**：由于 AI 生成耗时较长 (10-30s)，前端需设置较长的超时时间 (e.g., 60s)。

3.  **完成 (Completed)**
    - 请求返回，Loading 消失。
    - 展示结果卡片。

## 3. 技术实现
- **后端**：保持现有的同步接口 (`TopicController`)，无需 SSE 改造。
- **前端**：使用 React `useState` 控制 `loading` 状态。

## 4. 后续优化 (Optional)
- 未来可升级为“动态文案” (e.g., 每隔 5 秒切换一次文案："正在分析...", "正在撰写...")，以缓解用户等待焦虑，但无需后端配合。
